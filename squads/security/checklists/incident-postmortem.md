# Incident Postmortem Checklist

## Documentação do Incidente
- [ ] Título descritivo do incidente
- [ ] Severidade classificada (Critical/High/Medium/Low)
- [ ] Data e hora de detecção
- [ ] Data e hora de resolução
- [ ] Duração total do incidente

## Timeline
- [ ] Timeline completa com timestamps
- [ ] Quem detectou e como
- [ ] Ações de contenção tomadas
- [ ] Ações de erradicação tomadas
- [ ] Momento da recuperação

## Análise
- [ ] Root cause identificada
- [ ] Blast radius mapeado (dados/usuários afetados)
- [ ] Vetor de ataque documentado
- [ ] MITRE ATT&CK TTPs mapeados (se aplicável)

## Impacto
- [ ] Dados comprometidos quantificados
- [ ] Usuários afetados identificados
- [ ] Impacto no negócio avaliado
- [ ] Impacto regulatório avaliado (LGPD Art. 48)

## Comunicação
- [ ] Stakeholders internos notificados
- [ ] ANPD notificada (se dados pessoais, Art. 48)
- [ ] Titulares notificados (se necessário)
- [ ] Comunicação pública (se necessário)

## Lessons Learned
- [ ] O que funcionou bem
- [ ] O que falhou ou atrasou
- [ ] O que teria prevenido o incidente
- [ ] Action items com responsáveis e prazos

## Follow-up
- [ ] Vulnerabilidade corrigida
- [ ] Correção validada por @qa ou @red-team
- [ ] Monitoramento adicional implementado
- [ ] Action items do postmortem sendo acompanhados
